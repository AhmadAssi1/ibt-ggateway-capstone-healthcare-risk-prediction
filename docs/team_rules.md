# Team Rules

## 1. Responsibility
Each task must have one clear owner.

## 2. Communication
We use one main communication channel for updates and coordination.

## 3. Meetings
We meet at least once every week as a team.

## 4. Deadlines
We follow internal deadlines before the official bootcamp deadlines.

## 5. Progress
Each member must show weekly progress.

## 6. Blockers
If someone faces a problem or delay, they must report it early.

## 7. Scope
We focus on the core project first:
- cleaning
- EDA
- baseline model
- improved model
- evaluation
- README
- slides

## 8. Documentation
Important decisions must be documented clearly.

## 9. GitHub
The repository must stay organized and clean.

## 10. Teamwork
We communicate respectfully, support each other, and solve issues early.
